Glossary

Definitions of machine learning and system terms used in this panel

ML & System Terms
TermDefinitionContext
Batch Size Number of training samples processed together in one gradient update step. Smaller batches generalize better but train slower; larger batches are faster but may overfit. Model
Dropout Rate Fraction of neurons randomly disabled during each training step. Acts as regularization — forces the network to learn redundant representations and reduces overfitting. Model
Epoch One complete pass through the entire training dataset. More epochs allow the model to learn more, but too many cause overfitting on training data. Training
Feature A single measurable input variable fed to the model (e.g. area, number of rooms). The quality and relevance of features directly determines model accuracy. Data
Feature Count Total number of input features the model expects. Must match the length of every feature vector passed at inference time — mismatches cause runtime errors. Model
Feature Vector An ordered array of numeric feature values representing a single data record. Each position must correspond to the same feature every time. Data
Hidden Units Neurons in the intermediate layers of the neural network (not input or output). More units increase model capacity but also training time and overfitting risk. Configured as a comma-separated list per layer, e.g. 32,16. Model
Inference Using a trained model to generate a prediction on new, unseen input. Distinct from training — no weights are updated during inference. Model
Learning Rate Step size used when updating model weights during gradient descent. Too high causes instability; too low causes slow convergence or getting stuck in local minima. Typical range: 0.0001 – 0.01. Training
MAE Mean Absolute Error — average of the absolute differences between predicted and actual values. Intuitive unit: same scale as the target variable (e.g. price in PLN). Metrics
Metadata A JSON object attached to each record that stores the raw input values (features) before they are converted to a feature vector. Stored in PostgreSQL as a JSONB column. Data
Model A mathematical function — here a neural network built with TensorFlow.js — trained to map input feature vectors to numeric output predictions. Model
Num Samples Number of synthetic training examples generated when no real dataset is supplied. Higher values produce a smoother loss landscape but increase training time. Training
Overfitting When the model memorizes training data instead of learning general patterns. Symptoms: low training loss, high validation loss. Mitigated by dropout, fewer epochs, or more training data. Training
RMSE Root Mean Squared Error — square root of the average squared prediction errors. Penalizes large errors more than MAE does. Lower is better. Metrics
Tags String labels assigned to a record for grouping and filtering (e.g. apartment, house). Stored as a PostgreSQL text array with a GIN index for fast lookups. Data
Validation Rule-based checking of incoming record metadata before it is accepted. Enforces required fields, numeric types, integer constraints, and min/max ranges defined in the config. System
CSV Import Bulk-loading of records from a comma-separated values file. Column names are mapped to feature keys; the first row is optionally treated as a header. System
Measure One input parameter of the feature vector (e.g. area, rooms, floor). Each measure is individual and carries its own attributes — mean, deviation, slope and correlation with price. Model
Measure Strength How strongly an input parameter drives the predicted price, measured as the absolute correlation (|r|, 0–1) between that measure and the target. The model's total strength is the sum of the strengths of the measures it kept. Model
Slope & Deviation A measure's slope is the price change per unit of that input; its deviation is the spread of its values. Each measure is differentiated from the previous one in the hierarchy by its slope and deviation deltas. Model
Purity / Dirtiness Data quality of a measure: purity is the completeness of its source column, dirtiness is the missing fraction. A measure dirtier than the threshold (default 80%) is neutralised so it cannot alter the model. Data
Integration Hierarchy Measures are integrated into the model from the purest to the dirtiest. The cumulative strength column shows the running sum as each measure is added in that order. Model
Business Context A required plain-language description of what a model is for and how its output should be used. A model with no business context cannot be trained. System
